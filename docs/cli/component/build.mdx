# arcctl component build

Build a component and its container images into OCI artifacts.

## Synopsis

```bash
arcctl component build <path> -t <repo:tag> [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `<path>` | Path to the component directory (containing `architect.yml`) |

## Options

| Option | Description |
|--------|-------------|
| `-t, --tag <repo:tag>` | Tag for the root component artifact (required) |
| `--artifact-tag <name=repo:tag>` | Override tag for a specific child artifact (repeatable) |
| `-f, --file <path>` | Path to architect.yml if not in default location |
| `--platform <platform>` | Target platform (`linux/amd64`, `linux/arm64`) |
| `--no-cache` | Disable build cache |
| `--yes, -y` | Non-interactive mode (skip confirmation) |
| `--dry-run` | Show what would be built without building |

## Child Artifact Naming

When building a component, arcctl creates multiple artifacts:

```
# Root artifact
myregistry.io/myorg/myapp:v1.0.0

# Child artifacts (auto-generated names)
myregistry.io/myorg/myapp-deployment-api:v1.0.0
myregistry.io/myorg/myapp-deployment-worker:v1.0.0
myregistry.io/myorg/myapp-function-web:v1.0.0
myregistry.io/myorg/myapp-migration-main:v1.0.0
myregistry.io/myorg/myapp-cronjob-cleanup:v1.0.0
```

## Examples

```bash
# Basic build
arcctl component build . -t ghcr.io/myorg/myapp:v1.0.0

# Build with platform specification
arcctl component build . -t ghcr.io/myorg/myapp:v1.0.0 --platform linux/amd64

# CI/CD build (non-interactive)
arcctl component build . -t ghcr.io/myorg/myapp:$CI_COMMIT_SHA --yes

# Override specific child artifact tag
arcctl component build . \
  -t ghcr.io/myorg/myapp:v1.0.0 \
  --artifact-tag deployment/api=ghcr.io/myorg/custom-api:latest

# Build from specific file
arcctl component build ./services/api -f architect.production.yml -t ghcr.io/myorg/api:v1.0.0
```
