# cldctl create environment

Create a new environment.

<Note>
Use `cldctl create env` as shorthand for `cldctl create environment`.
</Note>

## Synopsis

```bash
cldctl create environment <name> [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `<name>` | Environment name |

## Options

| Option | Description |
|--------|-------------|
| `-d, --datacenter <name>` | Target datacenter (resolved from flag, `CLDCTL_DATACENTER` env var, or CLI config default) |
| `--if-not-exists` | Don't error if environment already exists |
| `--backend <type>` | State backend type |
| `--backend-config <key=value>` | Backend configuration |

## Examples

```bash
# Create an environment
cldctl create environment staging --datacenter aws-production

# Using alias
cldctl create env staging -d aws-production

# Create if not exists (idempotent)
cldctl create environment staging --datacenter aws-production --if-not-exists

# Create preview environment
cldctl create env preview-123 -d aws-staging
```

## Output

```
$ cldctl create env staging --datacenter aws-production

Environment: staging
Datacenter:  aws-production

[create] Creating environment "staging"...
[success] Environment created successfully
```
