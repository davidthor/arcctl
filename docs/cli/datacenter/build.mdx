# arcctl datacenter build

Build a datacenter and its IaC modules into OCI artifacts.

<Note>
Use `arcctl dc` as shorthand for `arcctl datacenter`.
</Note>

## Synopsis

```bash
arcctl datacenter build <path> -t <repo:tag> [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `<path>` | Path to the datacenter directory (containing `datacenter.hcl`) |

## Options

| Option | Description |
|--------|-------------|
| `-t, --tag <repo:tag>` | Tag for the root datacenter artifact (required) |
| `--module-tag <name=repo:tag>` | Override tag for a specific module (repeatable) |
| `-f, --file <path>` | Path to datacenter.hcl if not in default location |
| `--yes, -y` | Non-interactive mode |
| `--dry-run` | Show what would be built without building |

## Module Artifact Naming

When building a datacenter, arcctl bundles all IaC modules:

```
# Root artifact
myregistry.io/myorg/datacenter:v1.0.0

# Module artifacts (auto-generated names)
myregistry.io/myorg/datacenter-module-eks:v1.0.0
myregistry.io/myorg/datacenter-module-rds:v1.0.0
myregistry.io/myorg/datacenter-module-k8s-deployment:v1.0.0
```

## Examples

```bash
# Basic build
arcctl datacenter build . -t ghcr.io/myorg/aws-datacenter:v1.0.0

# Using alias
arcctl dc build . -t ghcr.io/myorg/aws-datacenter:v1.0.0

# CI/CD build
arcctl dc build . -t ghcr.io/myorg/aws-datacenter:$CI_COMMIT_SHA --yes

# Override module tag
arcctl dc build . \
  -t ghcr.io/myorg/aws-datacenter:v1.0.0 \
  --module-tag k8s=ghcr.io/myorg/custom-k8s:latest
```
