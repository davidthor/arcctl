# arcctl datacenter deploy

Deploy or update a datacenter.

<Note>
Use `arcctl dc` as shorthand for `arcctl datacenter`.
</Note>

## Synopsis

```bash
arcctl datacenter deploy <name> -c <config> [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `<name>` | Datacenter name |

## Options

| Option | Description |
|--------|-------------|
| `-c, --config <config-ref>` | Datacenter config: OCI image or local path (required) |
| `--var <key=value>` | Set a datacenter variable (repeatable) |
| `--var-file <path>` | Load variables from file |
| `--auto-approve` | Skip confirmation prompt |
| `--backend <type>` | State backend type |
| `--backend-config <key=value>` | Backend configuration |

## Examples

```bash
# Deploy from OCI image
arcctl dc deploy aws-production --config ghcr.io/myorg/aws-datacenter:v1.0.0

# Deploy from local source
arcctl dc deploy aws-production --config ./datacenters/aws

# Deploy with variables
arcctl dc deploy aws-production \
  --config ghcr.io/myorg/aws-datacenter:v1.0.0 \
  --var region=us-west-2 \
  --var cluster_name=prod-cluster

# Deploy with variable file
arcctl dc deploy aws-production \
  --config ghcr.io/myorg/aws-datacenter:v1.0.0 \
  --var-file ./vars/production.dcvars

# Deploy with auto-approval
arcctl dc deploy aws-production \
  --config ghcr.io/myorg/aws-datacenter:v1.0.0 \
  --auto-approve
```
