# arcctl datacenter tag

Create a new tag for an existing datacenter artifact and all its module artifacts.

<Note>
Use `arcctl dc` as shorthand for `arcctl datacenter`.
</Note>

## Synopsis

```bash
arcctl datacenter tag <source> <target> [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `<source>` | Source datacenter artifact reference |
| `<target>` | Target tag for the datacenter |

## Options

| Option | Description |
|--------|-------------|
| `--module-tag <name=repo:tag>` | Override tag for a specific module |
| `--yes, -y` | Non-interactive mode |

## Examples

```bash
# Tag a version as latest
arcctl dc tag ghcr.io/myorg/aws-datacenter:v1.0.0 ghcr.io/myorg/aws-datacenter:latest

# Tag for production
arcctl dc tag ghcr.io/myorg/aws-datacenter:v1.0.0 ghcr.io/myorg/aws-datacenter:production
```
