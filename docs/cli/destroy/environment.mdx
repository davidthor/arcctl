# cldctl destroy environment

Destroy an environment and all its resources.

<Warning>
This will destroy all components and resources in the environment. Use with caution.
</Warning>

<Note>
Use `cldctl destroy env` as shorthand for `cldctl destroy environment`.
</Note>

## Synopsis

```bash
cldctl destroy environment <name> [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `<name>` | Name of the environment to destroy |

## Options

| Option | Description |
|--------|-------------|
| `-d, --datacenter <name>` | Target datacenter (resolved from flag, `CLDCTL_DATACENTER` env var, or CLI config default) |
| `--auto-approve` | Skip confirmation prompt |
| `--backend <type>` | State backend type |
| `--backend-config <key=value>` | Backend configuration |

## Examples

```bash
# Destroy an environment (uses default datacenter)
cldctl destroy environment staging

# Using alias
cldctl destroy env staging

# Specify datacenter explicitly
cldctl destroy environment staging -d aws-production

# Auto-approve (CI/CD)
cldctl destroy environment preview-123 --auto-approve
```

## Output

```
$ cldctl destroy env staging

Environment: staging
Datacenter:  aws-production

This will destroy:
  - 3 components
  - 12 resources

Are you sure you want to destroy this environment? [y/N]:
```
