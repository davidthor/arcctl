# arcctl environment apply

Apply an environment configuration file.

<Note>
Use `arcctl env` as shorthand for `arcctl environment`.
</Note>

## Synopsis

```bash
arcctl environment apply -f <file> [options]
```

## Options

| Option | Description |
|--------|-------------|
| `-f, --file <path>` | Environment configuration file (required) |
| `--auto-approve` | Skip confirmation prompt |
| `--backend <type>` | State backend type |
| `--backend-config <key=value>` | Backend configuration |

## Examples

```bash
# Apply environment file
arcctl env apply -f staging.yml

# Apply with auto-approval
arcctl env apply -f staging.yml --auto-approve
```

## Environment File Format

```yaml
name: staging
datacenter: aws-production

locals:
  log_level: debug

components:
  web-app:
    component: ghcr.io/myorg/web-app:v1.5.0
    variables:
      log_level: ${{ locals.log_level }}
    scaling:
      api:
        replicas: 2
```

See [Environment Configuration](/environments/overview) for the complete file specification.
