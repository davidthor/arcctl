# cldctl images

List all component and datacenter artifacts in the local cache.

This is similar to `docker images` â€” it shows every artifact that has been
built locally or pulled from a remote registry.

## Synopsis

```bash
cldctl images [options]
```

## Options

| Option | Description |
|--------|-------------|
| `--type <type>` | Filter by artifact type: `component` or `datacenter` |
| `-o, --output <format>` | Output format: `table` (default), `json`, `yaml` |

## Description

The local artifact cache stores metadata about artifacts that have been:

- **Built locally** using `cldctl build component` or `cldctl build datacenter`
- **Pulled manually** using `cldctl pull component` or `cldctl pull datacenter`
- **Pulled implicitly** during deploy operations when the engine resolves an OCI reference

The cache is stored at `~/.cldctl/registry/artifacts.json` with artifact content
cached at `~/.cldctl/cache/artifacts/`.

When a build command is run without the `-t` flag, the artifact is identified by
its content digest (similar to `docker build` without `-t`). These entries appear
with `<none>` for both REPOSITORY and TAG.

## Examples

```bash
# List all cached artifacts
cldctl images

# List only components
cldctl images --type component

# List only datacenters
cldctl images --type datacenter

# JSON output (for scripting)
cldctl images -o json

# YAML output
cldctl images -o yaml
```

## Output

```
$ cldctl images

REPOSITORY                               TAG             ARTIFACT ID    TYPE         SOURCE     SIZE     CREATED
ghcr.io/myorg/web-app                   v1.0.0          a1b2c3d4e5f6   component    built      44.6MB   2 hours ago
docker.io/davidthor/startup-datacenter  latest          f6e5d4c3b2a1   datacenter   pulled     12.3KB   1 day ago
my-dc                                   latest          1234abcd5678   datacenter   built      8.4KB    3 days ago
```

### Table Columns

| Column | Description |
|--------|-------------|
| `REPOSITORY` | Repository portion of the artifact reference (or `<none>` for untagged builds) |
| `TAG` | Tag portion (or `<none>` for untagged builds) |
| `ARTIFACT ID` | Short content digest (first 12 hex characters) |
| `TYPE` | `component` or `datacenter` |
| `SOURCE` | `built` (locally) or `pulled` (from remote registry) |
| `SIZE` | Artifact size in human-readable format |
| `CREATED` | Relative time since the artifact was added |
