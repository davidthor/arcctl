# cldctl pull component

Pull a component artifact from an OCI registry to the local cache.

<Note>
Use `cldctl pull comp` as shorthand for `cldctl pull component`.
</Note>

## Synopsis

```bash
cldctl pull component <repo:tag> [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `<repo:tag>` | Component artifact reference |

## Options

| Option | Description |
|--------|-------------|
| `-q, --quiet` | Suppress output |

## Description

This command downloads the component artifact and registers it in the local
artifact registry. The component can then be used for deployment or inspection.
Once pulled, the component appears in `cldctl images`.

## Examples

```bash
# Pull a component
cldctl pull component ghcr.io/myorg/myapp:v1.0.0

# Using alias
cldctl pull comp ghcr.io/myorg/myapp:v1.0.0

# Pull with quiet mode
cldctl pull component docker.io/library/nginx:latest -q
```

## Output

```
$ cldctl pull component ghcr.io/myorg/myapp:v1.0.0

Pulling component: ghcr.io/myorg/myapp:v1.0.0
[pull] Downloading ghcr.io/myorg/myapp:v1.0.0...
[success] Pulled ghcr.io/myorg/myapp:v1.0.0
  Cache: /home/user/.cldctl/cache/artifacts/ghcr.io_myorg_myapp_v1.0.0
```
