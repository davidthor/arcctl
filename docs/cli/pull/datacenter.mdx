# cldctl pull datacenter

Pull a datacenter artifact from an OCI registry to the local cache.

<Note>
Use `cldctl pull dc` as shorthand for `cldctl pull datacenter`.
</Note>

## Synopsis

```bash
cldctl pull datacenter <repo:tag> [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `<repo:tag>` | Datacenter artifact reference |

## Options

| Option | Description |
|--------|-------------|
| `-q, --quiet` | Suppress output |

## Description

This command downloads the datacenter artifact and registers it in the local
artifact registry. The datacenter can then be used for deployment or inspection.

Once pulled, the datacenter appears in `cldctl images` and can be referenced by
tag when deploying, without requiring another network fetch.

## Examples

```bash
# Pull a datacenter
cldctl pull datacenter docker.io/davidthor/startup-datacenter:latest

# Using alias
cldctl pull dc docker.io/davidthor/startup-datacenter:latest

# Pull with quiet mode
cldctl pull datacenter ghcr.io/myorg/my-dc:v1.0.0 -q
```

## Output

```
$ cldctl pull datacenter docker.io/davidthor/startup-datacenter:latest

[pull] Downloading docker.io/davidthor/startup-datacenter:latest...
[success] Pulled docker.io/davidthor/startup-datacenter:latest
  Cache: /home/user/.cldctl/cache/artifacts/docker.io_davidthor_startup-datacenter_latest
```
