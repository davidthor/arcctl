# cldctl tag datacenter

Create a new tag for an existing datacenter artifact and all its module artifacts.

<Note>
Use `cldctl tag dc` as shorthand for `cldctl tag datacenter`.
</Note>

## Synopsis

```bash
cldctl tag datacenter <source> <target> [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `<source>` | Source artifact reference |
| `<target>` | Target artifact reference |

## Options

| Option | Description |
|--------|-------------|
| `--module-tag <name=repo:tag>` | Override tag for a specific module (repeatable) |
| `-y, --yes` | Non-interactive mode |

## Examples

```bash
# Tag an artifact
cldctl tag datacenter ghcr.io/myorg/dc:v1.0.0 ghcr.io/myorg/dc:latest

# Using alias
cldctl tag dc ghcr.io/myorg/dc:v1.0.0 ghcr.io/myorg/dc:latest

# Non-interactive
cldctl tag datacenter my-dc:dev my-dc:staging -y

# Override module tag
cldctl tag datacenter ghcr.io/myorg/dc:v1.0.0 ghcr.io/myorg/dc:latest \
  --module-tag k8s=ghcr.io/myorg/k8s:latest
```

## Output

```

  Target: ghcr.io/myorg/dc:latest

Proceed with tagging? [Y/n]:
[success] Tagged ghcr.io/myorg/dc:v1.0.0 as ghcr.io/myorg/dc:latest
```
