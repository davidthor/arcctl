# cldctl validate datacenter

Validate a datacenter configuration file without deploying.

<Note>
Use `cldctl validate dc` as shorthand for `cldctl validate datacenter`.
</Note>

## Synopsis

```bash
cldctl validate datacenter [path] [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `[path]` | Path to the datacenter directory or datacenter.hcl file (default: current directory) |

## Options

| Option | Description |
|--------|-------------|
| `-f, --file <path>` | Path to datacenter.hcl if not in default location |

## Examples

```bash
# Validate current directory
cldctl validate datacenter

# Using alias
cldctl validate dc

# Validate specific directory
cldctl validate datacenter ./my-datacenter

# Validate specific file
cldctl validate datacenter -f custom-datacenter.hcl
```

## Output

```
$ cldctl validate datacenter ./my-datacenter

Datacenter configuration is valid!
```

**On validation error:**

```
$ cldctl validate datacenter ./invalid-dc

Error: validation failed: [PARSE_ERROR] Invalid HCL syntax at line 15:
  unexpected token "{"
```
